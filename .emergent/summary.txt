<analysis>**original_problem_statement:**
O usuário iniciou o projeto para continuar o desenvolvimento do aplicativo **SoulNutri**, um sistema de identificação de pratos em tempo real para restaurantes. O objetivo é fornecer ao consumidor informações detalhadas (ingredientes, alérgenos, dados nutricionais) antes da escolha do prato.

**PRODUCT REQUIREMENTS (baseado nos documentos e na conversa):**
1.  **Visão:** Funcionar como um Waze para alimentação, com identificação de pratos por imagem em tempo real.
2.  **Core Feature:** Resposta da API de identificação em ~100ms.
3.  **Fluxo de Fallback:** Imagem -> OCR -> QR Code.
4.  **Política de Confiança:** Usar scores para determinar a confiança (ALTA/MÉDIA/BAIXA).
5.  **Dados da Resposta:** A API deve retornar nome, descrição, ingredientes, benefícios, riscos, informações nutricionais, categoria e técnicas de preparo.
6.  **Dados de Origem (Premissas Cibi Sana):**
    *   Todos os pratos devem refletir as práticas culinárias do Cibi Sana: sem aditivos químicos, ingredientes orgânicos/frescos, técnicas específicas (sous vide, vapor, etc.), molhos artesanais, especiarias importadas.
    *   O texto sem aditivos químicos e/ou alimentos industrializados/processados deve ser exibido.
    *   A lógica de glúten deve ser precisa (se um prato contém glúten, não deve avisar sobre traços).
7.  **UX/UI:**
    *   Interface com o mínimo de cliques, câmera contínua e modo de auto-captura funcional.
    *   A posição da câmera e o logo da Emergent não devem atrapalhar a usabilidade.
    *   Deve haver um botão para limpar o resultado e capturar uma nova foto.
