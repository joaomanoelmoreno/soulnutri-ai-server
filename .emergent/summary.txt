<analysis>**original_problem_statement:**
O usuário iniciou o projeto para continuar o desenvolvimento do aplicativo **SoulNutri**, um sistema de identificação de pratos em tempo real para restaurantes. O objetivo é fornecer ao consumidor informações detalhadas (ingredientes, alérgenos, dados nutricionais, fatos científicos relevantes) antes da escolha do prato, funcionando como um agente de nutrição virtual.

**PRODUCT REQUIREMENTS (baseado nos documentos e na conversa):**
1.  **Visão:** Funcionar como um agente de nutrição virtual ou Waze da alimentação, com identificação de pratos por imagem em tempo real. O slogan a ser lembrado é: SOULNUTRI - O SEU AGENTE DE NUTRICAO VIRTUAL.
2.  **Core Feature:** Identificação de pratos por imagem com resposta rápida. O objetivo inicial de 100ms foi flexibilizado, com ~250ms sendo aceitável em ambiente de CPU.
3.  **Fluxo de Identificação:**
    *   **Nível 1 (Índice Local):** Tentar identificar o prato no banco de dados local (pratos do Cibi Sana) via OpenCLIP. A meta é 100% de acerto para pratos cadastrados.
    *   **Nível 2 (IA Genérica):** Se a confiança for baixa ou o prato não for encontrado, usar uma IA genérica (GPT-4o Vision) para uma identificação provisória.
    *   **Nível 3 (Fallback):** OCR e QR Code são funcionalidades futuras.
4.  **Qualidade da Informação:**
    *   As informações devem ser educativas, cientificamente embasadas e despertar curiosidade, evitando dados óbvios. Ex: explicar *por que* o potássio é bom, não apenas listá-lo.
    *   Deve haver um equilíbrio entre notícias boas e alertas, para não ser um app do apocalipse.
    *   Para alertas de risco (ex: mercúrio no atum), deve haver um link para uma notícia ou estudo relevante.
5.  **Dados da Resposta:** A API deve retornar nome, descrição, ingredientes, categoria (vegano, etc.), **benefício principal**, **alerta/curiosidade principal**, e um **fato científico com link para fonte**.
6.  **UX/UI:**
    *   Interface com o mínimo de cliques, com captura por toque na tela.
    *   Câmera com moldura guia retangular e vertical para enquadramento.
    *   Sistema de feedback para o usuário:
        *   Confirmar que a identificação está correta.
        *   Corrigir a identificação, selecionando o prato certo de uma lista.
        *   Cadastrar um prato totalmente novo, cujo nome será usado por uma IA para gerar todas as informações.
        *   Salvar fotos (mesmo as ruins/tremidas) para treinar a IA.
        *   Descartar fotos inutilizáveis.
7.  **Escalabilidade:** O sistema deve permitir que múltiplos usuários testem simultaneamente e que os dados (pratos, imagens de feedback) sejam centralizados.

**User's preferred language**: Português (Brasil). O próximo agente DEVE se comunicar em Português.
