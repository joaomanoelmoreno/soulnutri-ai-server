## PROBLEMA ATUAL (PRIORIDADE MÁXIMA)

### Reconhecimento CLIP com precisão muito baixa
- **Sintoma:** Pratos do Cibi Sana sendo reconhecidos com ~50-60% de score, quando ontem estavam com 90%+
- **Causa identificada:** A calibração de confiança está derrubando scores altos para 50% quando o "gap" entre 1º e 2º lugar é pequeno
- **Localização:** `/app/backend/ai/index.py` linhas 240-250

**Exemplo do problema:**
```
Raw score: 0.888 (88.8% - muito bom!)
Gap: 0.010 (pequeno)
Calibrated: 0.50 (50% - ERRADO!)
```

**Código problemático (linha 242-243):**
```python
if gap < 0.02:
    calibrated_score = min(top1_score * 0.6, 0.50)  # Derruba tudo para 50%!
```

### Solução recomendada
A calibração foi projetada para scores brutos de 0.15-0.35, mas o índice atual retorna scores de 0.85-0.95. Precisa ajustar para considerar scores brutos altos (>0.80) como confiáveis mesmo com gap pequeno.
